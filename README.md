# Securities-Hero

증권 히어로 입니다.

저희는 팀 프로젝트로 진행했으며, IPO 절차의 증권 신고서 초안을 작성하고 검토하는 AI 기능 웹 사이트를 제작했습니다.

저는 백엔드 프론트와 AI 보조를 맡았습니다.

백엔드의 로그인, 증권 신고서 페이지와 프론트-백-Kafka-AI 연동 로직을 만들었으며, FastAPI 서버의 RAG 기반 LLM과 Ai 서버의 랭 그래프 제작을 지원했습니다.

프론트는 모든 페이지를 제작했습니다.

로그인은 JWT기반  Stateless로 제작했습니다. 

또한 MCP 서버를 제작하지는 못했지만 사용하기 위해 MicroService를 사용할 수 있게 제작했습니다. 


증권 신고서를 사용자가 작성하겠다. 초안 생성 버튼을 누르면

프론트에서 백엔드로 요청을 보내며 해당 증권 신고서와 회사의 내용을 RDB와 OpenSearch에서 가져와 데이터를 매핑합니다.

백엔드에서 각각의 AI 서버로 Kafka 요청을 보내 응답을 받아서 프론트에서 요청한 내용에 맞게 가공한 뒤 프론트로 전송합니다.

여기서 AI의 응답이 오래 걸리기 때문에 서버와 Http의 요청 제한시간 문제를 해결하기 위해 Kafka를 사용하게 되었습니다.
